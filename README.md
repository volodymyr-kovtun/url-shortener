# URL Shortener

[![CI](https://github.com/volodymyr-kovtun/url-shortener/actions/workflows/ci.yml/badge.svg)](https://github.com/volodymyr-kovtun/url-shortener/actions/workflows/ci.yml)
[![.NET](https://img.shields.io/badge/.NET-10.0-512BD4)](https://dotnet.microsoft.com/en-us/download/dotnet/10.0)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](./LICENSE)

A small, production-shaped ASP.NET Core service that turns long URLs into short
slugs and redirects them back. Storage is pluggable: SQLite is the default
zero-setup local backend, Azure Table Storage is bundled, and adding another
backend is a new project plus a one-line dispatch entry â€” see
[Adding a storage provider](#adding-a-storage-provider).

## Features

- **Pluggable storage** behind a single `IStorageProvider<UrlMapping, string>`
  abstraction â€” SQLite (EF Core) and Azure Table Storage are bundled.
- **API key authentication** on the create endpoint via an `X-Api-Key` header;
  resolution is public.
- **Hot-path caching** with [`HybridCache`](https://learn.microsoft.com/en-us/aspnet/core/performance/caching/hybrid)
  on URL resolution to absorb repeat traffic without hitting the backing store.
- **OpenAPI 3.1** generated by the built-in `Microsoft.AspNetCore.OpenApi`
  pipeline, served at `/openapi/v1.json` with a [Scalar](https://scalar.com)
  UI at `/scalar/v1`.
- **Centralized error handling** via `IExceptionHandler` returning RFC 7807
  `ProblemDetails`.
- **Fail-fast configuration** â€” settings are validated at startup with
  `OptionsBuilder.ValidateDataAnnotations().ValidateOnStart()`, so a missing
  API key or connection string aborts the host before the first request.
- **Docker-first** local development with a SQLite default and an Azurite
  profile for the Azure Table Storage backend.

## Quick start

### Docker (recommended)

The fastest way to run the service locally is via `docker compose`. The default
profile uses SQLite stored in a Docker volume â€” no external services required.

```bash
cp docker/.env.example docker/.env
# edit docker/.env and set API_KEY to a long random string
docker compose -f docker/docker-compose.yaml --env-file docker/.env up --build
```

Then open <http://localhost:8080/scalar/v1> for the API reference.

To exercise the Azure Table Storage backend locally via the
[Azurite](https://github.com/Azure/Azurite) emulator instead:

```bash
docker compose -f docker/docker-compose.azure.yaml --env-file docker/.env up --build
```

### Without Docker

1. Install the [.NET 10 SDK](https://dotnet.microsoft.com/en-us/download/dotnet/10.0).
2. Configure the API key as a user-secret:

   ```bash
   dotnet user-secrets init --project src/UrlShortener.Host
   dotnet user-secrets set "Auth:ApiKey" "your-dev-key" --project src/UrlShortener.Host
   ```

3. Run the API. SQLite is the default â€” a `urlshortener-dev.db` file is created
   in the working directory on first startup:

   ```bash
   dotnet run --project src/UrlShortener.Host
   ```

   The Scalar API reference is at <http://localhost:5194/scalar/v1>.

## Using the API

Create a short URL (requires the API key in the `X-Api-Key` header):

```bash
curl -X POST http://localhost:8080/api/shorten \
  -H "Content-Type: application/json" \
  -H "X-Api-Key: $API_KEY" \
  -d '{"url":"https://example.com/some/very/long/path","prefix":"demo"}'
```

```json
{
  "originalUrl": "https://example.com/some/very/long/path",
  "shortUrl": "http://localhost:8080/demo-1a2b3c4d"
}
```

`prefix` is optional. When omitted the short code is just the 8-character slug.

Resolve a short URL (no auth required) â€” issues a `302` redirect:

```bash
curl -i http://localhost:8080/demo-1a2b3c4d
```

### Error responses

All errors return RFC 7807 `application/problem+json`:

```json
{
  "type": "https://tools.ietf.org/html/rfc9110#section-15.5.1",
  "title": "One or more validation errors occurred.",
  "status": 400,
  "errors": { "$": ["JSON deserialization for type 'CreateShortUrlRequest' was missing required properties including: 'url'."] }
}
```

| Status | When |
| --- | --- |
| `200` | Short URL created |
| `302` | Resolve hit â€” `Location` header points at the original URL |
| `400` | Invalid request body (missing `url`, malformed JSON, non-absolute URL) |
| `401` | Missing or invalid `X-Api-Key` on `POST /api/shorten` |
| `404` | Resolve miss â€” no mapping for the given short code |
| `500` | Unhandled exception â€” logged and returned as `ProblemDetails` |

## Configuration

All settings can be supplied via `appsettings.json`, environment variables, or
user-secrets. The double-underscore form is required for environment variables
(`Auth__ApiKey`, `Storage__Provider`, etc.). Each required key is validated on
host startup; a missing or empty value aborts the process with a descriptive
error.

| Key | Required | Default | Description |
| --- | --- | --- | --- |
| `Auth:ApiKey` | yes | â€” | Shared secret required in the `X-Api-Key` header on `POST /api/shorten` |
| `Shortener:Domain` | yes | â€” | Base URL prepended to generated short codes in the response |
| `Storage:Provider` | no | `Sqlite` | Backend to use: `Sqlite` or `AzureTableStorage` |
| `Storage:Sqlite:ConnectionString` | when provider is `Sqlite` | â€” | EF Core SQLite connection string, e.g. `Data Source=urlshortener.db` |
| `Storage:AzureTableStorage:ConnectionString` | when provider is `AzureTableStorage` | â€” | Azure Storage account connection string |

## Architecture

The composition root lives in
[`Host/Storage/StorageRegistration.cs`](src/UrlShortener.Host/Storage/StorageRegistration.cs).
It reads `Storage:Provider` and dispatches to the matching
`Add<Provider>Storage(IConfiguration)` extension method exposed by each
infrastructure project. Each provider owns its own settings type, its own
connection lifecycle, and its own bootstrap concerns (the SQLite provider
registers a hosted service that runs `EnsureCreated` on startup).

Dependencies flow inward: `Host` and `Infrastructure.*` depend on `Core`,
never the reverse. Application code only ever touches
`IStorageProvider<UrlMapping, string>` â€” the concrete backend is invisible
to the controller and service layer.

### Adding a storage provider

1. Create `src/UrlShortener.Infrastructure.<Name>Storage/` with:
   - An implementation of `IStorageProvider<UrlMapping, string>`.
   - A `<Name>StorageSettings` class with data-annotation validation
     (`[Required]`, etc.) for its configuration.
   - A `<Name>StorageRegistry` static class exposing
     `Add<Name>Storage(this IServiceCollection, IConfiguration)` (bind settings
     via `AddOptions<T>().Bind(...).ValidateDataAnnotations().ValidateOnStart()`)
     and a `ProviderName` constant matching the value users set in
     `Storage:Provider`.
2. Add a `ProjectReference` from `UrlShortener.Host` to the new project.
3. Add a case to the switch in
   [`Host/Storage/StorageRegistration.cs`](src/UrlShortener.Host/Storage/StorageRegistration.cs).
4. Document the new `Storage:<Name>:*` keys in the [Configuration](#configuration)
   table above.

No other code needs to change â€” the domain, controllers, and existing providers
are untouched.
